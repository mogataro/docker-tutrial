Docker用語集

Dockerイメージ
Dockerコンテナを作成するための雛形。
Dockerイメージは、アプリケーション、ライブラリ、設定ファイルなどのアプリケーション実行に必要なもの一式をまとめたもの。
出来上がったアプリケーションをDockerイメージとして保存して、別のサーバーに持っていくことで同じ環境（コンテナ）を別のサーバー上で再現することができる。
Dockerコンテナ
Dockerイメージを元に作成されるコンテナ型仮想環境のことをDockerコンテナ、または単にコンテナと呼ぶ。
イメージからコンテナを作成することで、何度でも簡単に同じコンテナ（仮想環境）を作成することができる。
コンテナを起動することで、予めイメージにセットアップしたアプリケーションの機能を提供することができる。
Docker Hub
Dockerイメージを保存するための機能などを提供しているサービス。
ベンダーや他のユーザーが作ったイメージも公開されており、公開されているイメージをダウンロードすることで様々なコンテナを起動することができる。
また、Docker Hubの様にイメージを保管するための機能を持ったものをレジストリと呼ぶ。
nginx
Webサーバーのソフトウェア
Apache
Webサーバーのソフトウェア
MySQL
データベースサーバーのソフトウェア。
RDBMS（リレーショナルデータベースマネジメントシステム）の一種。
PostgreSQL
データベースサーバーのソフトウェア。
RDBMS（リレーショナルデータベースマネジメントシステム）の一種。
管理ツールとしてpgAdminなどがある。
Ubuntu
Linux OSのディストリビューションの1つ。
Redis
Key-Value型でデータを保存するキーバリューストアと呼ばれるデータベースシステムの一種。
メモリ上にデータを保持して高速に動作する。
アプリケーションのデータのキャッシュや、セッション情報の保存によく使用される。
Node.js
Javascriptの実行環境で、サーバー上でJavascriptのプログラムを実行することができる。
ホストOS
仮想化において、仮想サーバーを起動させる側のOSのこと。
ホストOS上に仮想化のソフトウェアをインストールすることで、仮想マシンを作成して複数の仮想環境を起動することができる。
ゲストOS
仮想化において、ホストOS上に作成された仮想マシン上で動作するOSのこと。
Virtual Machine（VM）
仮想環境のことをVirtual Machineや略してVMと呼ぶ。
一般的にはホスト型仮想化の仮想マシンのことを指す。
ホスト型仮想化
ホストOSにハイパーバイザーと呼ばれる仮想化の機能を用意して行う仮想化のことをホスト型仮想化と言う。
ハイパーバイザーの機能を提供するソフトウェアとしては、Virtual Box、Hyper-V、VMwareなどがある。
ホストOSに依存せずに仮想マシンに好きなOSをインストールできる点や、仮想マシン間の分離レベルが高い点などがコンテナ仮想化と異なる。
コンテナ型仮想化
Dockerといったコンテナ仮想化を行うソフトウェアを用いて行う仮想化をコンテナ型仮想化と言う。
Dockerの場合はDockerエンジンによってコンテナの作成、削除などの管理が行われる。
ホスト型とは異なり、ゲストOSというものはなく、ホストOSのカーネルを共有してアプリケーションが実行される。
ハイパーバイザー
ホスト型仮想化の仮想マシンを管理するための機能を提供するもの。
ホストOSとゲストOSの間を仲介する形でハイパーバイザーが機能する。
Dockerエンジン
Dockerにおいてコンテナ型仮想化を実現するためのコアとなる機能を持った部分。
Dockerエンジンによってコンテナ作成などの機能が提供される。
オーバーヘッド
コンピュータで何らかの処理を行う際に、その処理を行うために余計に費やされるシステムへかかる負荷や処理時間などのことを言う。
ミドルウェア
コンピュータの基本的な制御を行うオペレーティングシステムと、各業務処理を行うアプリケーションソフトウェアとの中間に入るソフトウェアのこと。
Webアプリケーションでは、DBサーバーやWebサーバー、アプリケーションサーバーの機能を提供するミドルウェアなどが使われる。
カーネル
OSのコアとなる部分。
ハードディスクやメモリーなど、コンピューターリソースの管理や、プログラムの実行プロセスへのリソースの割り当てなどを行う。
パッケージ
特定の機能を提供するために、必要なプログラムをまとめたもの。
Linuxサーバーなどでは、公開されている様々なパッケージをインストールすることで、サーバーとしての機能を追加することができる。
またプログラミングでは、他者が公開しているパッケージを再利用することで、全てを自前で作成せずに部分的にパッケージの機能を利用してアプリケーションを作成することができる。
